Primeiro passo: Criando um repositório no GitHub

O VS Code torna o uso do Git muito visual e intuitivo. Como você já tem o Git instalado, o VS Code vai reconhecê-lo automaticamente.
Aqui está o guia passo a passo para você dominar o GitHub sem precisar sair do editor:

1. Conectar seu VS Code ao GitHub
Antes de tudo, o VS Code precisa de permissão para falar com sua conta:

Clique no ícone de Controle de Origem (parece um nó de árvore) na barra lateral esquerda (ou aperte Ctrl + Shift + G).

Clique em "Publish to GitHub" (se for um projeto novo) ou procure o ícone de conta no canto inferior esquerdo do VS Code para fazer o Sign-in.

2. Criar um Repositório Novo (Direto pelo VS Code)
Você não precisa ir ao site do GitHub para criar o repositório. Siga este fluxo:

Abra a pasta do seu projeto no VS Code (File > Open Folder).

Vá na aba de Controle de Origem (Ctrl + Shift + G).

Clique no botão "Initialize Repository". Isso é o equivalente ao git init.

Após inicializar, aparecerá o botão "Publish Branch".

O VS Code vai te dar duas opções no topo da tela:

Publish to GitHub private repository: Só você vê.

Publish to GitHub public repository: Fica no seu portfólio para todos verem.

Escolha uma, e pronto! O repositório será criado e o código enviado automaticamente.

3. O Ciclo de Trabalho (Add, Commit, Push Visual)
Sempre que você alterar um arquivo, ele aparecerá na aba de Controle de Origem com a letra M (Modified).

Stage (O git add): Ao lado do nome do arquivo alterado, clique no ícone de "+". O arquivo subirá para a seção "Staged Changes".

Commit: Na caixa de texto acima, escreva sua mensagem (ex: "Criação da estrutura HTML"). Depois, clique no botão azul "Commit".

Push: Após o commit, aparecerá um botão escrito "Sync Changes" (Sincronizar Alterações) com um ícone de nuvem ou setas. Clique nele. Isso fará o push para o GitHub.

4. Clonar um Repositório (git clone)
Se você quer baixar um projeto seu ou de outra pessoa:

Pressione Ctrl + Shift + P para abrir a paleta de comandos.

Digite "Git: Clone" e aperte Enter.

Cole a URL do repositório (ex: https://github.com/usuario/projeto.git).

Escolha a pasta no seu computador onde o projeto será salvo.

O VS Code perguntará se você quer abrir o repositório clonado. Clique em "Open".

5. Dica para ser mais produtivo: O Terminal Integrado
Embora os botões facilitem, às vezes o terminal é mais rápido. Você pode abrir o terminal dentro do VS Code com Ctrl + ' (crase).

Dica Prática: Se você estiver desenvolvendo em JavaScript ou Java, lembre-se de sempre criar um arquivo chamado .gitignore na pasta raiz. Escreva o nome das pastas que você não quer que subam para o GitHub (como node_modules/ ou arquivos de configuração pessoal). O VS Code deixará esses arquivos "cinzas", indicando que o Git vai ignorá-los.
