Primeiro passo: Criando um repositório no GitHub

O VS Code torna o uso do Git muito visual e intuitivo. Como você já tem o Git instalado, o VS Code vai reconhecê-lo automaticamente.

Aqui está o guia passo a passo para você dominar o GitHub sem precisar sair do editor:

1. Conectar seu VS Code ao GitHub

Antes de tudo, o VS Code precisa de permissão para falar com sua conta:

  • Clique no ícone de Controle de Origem na barra lateral esquerda (ou aperte Ctrl + Shift + G).
  • Clique em "Publish to GitHub" ou procure o ícone de conta no canto inferior esquerdo para fazer o Sign-in.

2. Criar um Repositório Novo

  1. Abra a pasta do seu projeto no VS Code (File > Open Folder).
  2. Vá na aba de Controle de Origem.
  3. Clique em Initialize Repository.
  4. Após inicializar, selecione Publish Branch.

3. O Ciclo Diário: O "ritual" de alimentar o GitHub

Para manter seu repositório atualizado, siga este ciclo simples:

  • Adicionar: Faça suas mudanças no código e depois vá para a aba de Controle de Origem. Você verá os arquivos modificados listados. Clique no ícone de "+" para adicionar as mudanças ao stage.
  • Commitar: Escreva uma mensagem clara e descritiva sobre o que você mudou (ex: "Adiciona função de login") e clique em "Commit".
  • Enviar para o GitHub: Depois de commitar, clique em "Push" para enviar suas mudanças para o repositório remoto no GitHub.
Dica Prática: Se você estiver desenvolvendo em JavaScript, lembre-se do arquivo .gitignore.

Sumário de HTML

1. Fundamentos e Sintaxe

  • O que é uma tag: Anatomia de um elemento (tag de abertura, conteúdo e tag de fechamento).
  • Atributos: Como passar informações extras para as tags (ex: href, src, id, class).
  • Estrutura Global: As tags obrigatórias que todo arquivo deve ter (!DOCTYPE html, html, head, body).

2. Organização de texto

  • Titulos (Headings) A hierarquia de h1 a h6.
  • Parágrafos e Quebras: Uso de p, br e hr.
  • Formatação Básica: Negrito (strong), itálico (em) e citações.
  • Listas: Listas ordenadas (ol), não ordenadas (ul) e itens de lista (li).

3. Hiperlinks e multimídia

  • Âncoras (a): Criando links internos, externos e links para e-mail/WhatsApp.
  • Imagens (img): Inserção correta e a importância do atributo alt (acessibilidade).
  • Vídeo e Áudio: Como incorporar mídia nativa.

4. Estruturação e Semântica (O Coração do HTML Moderno)

  • Tags Semânticas: Por que usar header, nav, main, section, article, aside e footer em vez de apenas div.
  • Acessibilidade e SEO: Como a estrutura correta ajuda o Google e leitores de tela.

5. Tabelas e Formulários

  • Tabelas: Estrutura de dados com table, tr, th e td.
  • Formulários(forms): Captura de dados com input, textarea, select, button e label.
